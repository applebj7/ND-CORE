# ND-CORE 프로그램 사양서 및 시스템 아키텍처

## 1. 개요
ND-CORE는 로컬 환경의 파일 시스템을 기반으로 빠르고 정확한 파일 검색 및 내용 기반(시멘틱) 검색을 제공하는 경량화된 웹 애플리케이션입니다. 직관적인 UI(Glassmorphism 기반 다크 테마)와 더불어 Python 기반의 유연한 문서 및 이미지 전처리 모듈을 탑재하여 텍스트부터 Office 문서, 이미지 내 글자까지 폭넓게 탐색할 수 있습니다.

---

## 2. 시스템 아키텍처
본 시스템은 **Python(Flask) 백엔드**와 **Vanilla JS / HTML / CSS 프론트엔드**로 구성된 클라이언트-서버 구조로 동작하며, 별도 웹서버 없이 로컬에서 바로 실행 및 접근(127.0.0.1:5000)이 가능합니다.

### 2.1 Backend (백엔드)
- **프레임워크:** Python 3.x, Flask, Flask-CORS
- **수행 역할:**
  - 로컬 파일 시스템 순회 및 검색 (`os.walk` 활용)
  - 확장자별 텍스트 전처리 및 파싱
  - 정적 웹 자원(프론트엔드 HTML/CSS/JS) 호스팅 (`/` 라우트)
  - 검색된 폴더/파일의 윈도우 탐색기 열기 (`subprocess.Popen` 활용)

### 2.2 Frontend (프론트엔드)
- **사용 기술:** HTML5, Vanilla JavaScript (ES6), CSS3, FontAwesome
- **수행 역할:**
  - 비동기식(Fetch API) 화면 전환 (메인 화면 ↔ 파일 검색 ↔ 링크 모음)
  - 모던 UI 제공 (Glassmorphism, Ambient Orbs 에셋, 반응형 레이아웃)
  - 검색 결과 데이터 비동기 호출 및 랜더링 결과 제공
  - 검색어 일부분 및 요약 스니펫(Snippet) 내부의 하이라이팅 처리 (RegEx 사용)

## 3. 주요 기능 및 사양

### 3.1 파일 이름 및 확장자 검색
- 검색어 필드에서 `이름` 조건을 선택할 경우 수행됩니다.
- 입력된 검색어가 파일명 내에 부분 문자열로 포함(**Substring Match**)되거나, 파일의 확장자와 정확히 일치(**Exact Match**)하는 파일을 찾아 결과 목록으로 반환합니다.

### 3.2 내용 검색 및 텍스트 파싱 로직 (시멘틱 검색 임시 구현)
- 조건이 `내용(content)`인 경우, 파일 내부의 텍스트를 파싱하여 검색어가 포함되어 있는지 검사합니다.
- 단순 일치를 넘어, 일치하는 구문을 중심으로 **위/아래 1줄을 포함한 3줄 분량의 요약 스니펫(Snippet)** 을 추출하여 사용자에게 제공합니다.
- **지원 파일 및 파싱 기술:**
  1. **일반 텍스트 파일** (`.txt`, `.md`, `.py`, `.json`, `.csv`, `.html` 등): 기본 텍스트 읽기
  2. **Office 문서 (Word):** `.docx` 문서의 단락(Paragraph) 텍스트 추출 (`python-docx` 사용)
  3. **Office 문서 (PPT):** `.pptx` 문서 내 슬라이드 및 도형 텍스트 추출 (`python-pptx` 사용)
  4. **Office 문서 (Excel):** `.xlsx` 문서 내 활성 시트의 셀 값 추출 (`openpyxl` 사용)
  5. **PDF 문서:** `.pdf` 파일의 페이지별 텍스트 파싱 (`PyPDF2` 사용)
  6. **이미지 파일:** `.png`, `.jpg`, `.jpeg`, `.bmp` 파일에 인쇄된 글자 추출 (`paddleocr` 광학 문자 인식(OCR) 적용)
> **안전장치:** 사용자의 PC에 해당 라이브러리(PPOCR, docx 등)가 설치되어 있지 않더라도, `ImportError`를 감지하여 서버가 다운되지 않도록 Graceful Degradation 전략이 적용되어 있습니다.

### 3.3 사용자 인터페이스(UI) 특징
- **동적 라우팅:** `search_ui.html`을 베이스 템플릿으로 삼아, 내부 `<div id="viewContainer">` 영역에 `fileSearch.html`, `link.html`을 동적으로 Fetch하여 깜빡임 없는 전환을 구현합니다.
- **데이터 식별성:** CSS `line-clamp: 3`과 `text-overflow: ellipsis`를 활용하여 추출된 스니펫이 3줄을 초과할 경우 말줄임표 처리하여 가독성을 높였습니다. 
- **색상 토큰:** 확장자 종류(문서, 코드, 미디어, 압축파일 등)를 파악해 아이콘 별로 다른 테마 색상(파랑, 초록, 오렌지, 빨강 색상 등)을 부여합니다.

## 4. 구동 방식 및 폴더 구조

### 4.1 실행 환경
1. Python 전제 조건 및 라이브러리 설치 (`flask`, `flask-cors` 및 파싱 기술에 필요한 OCR 및 Office 라이브러리)
2. `c:\PJT_new>` 경로 등에서 터미널을 열고 `python app.py` 커맨드 입력
3. `http://127.0.0.1:5000`으로 브라우저 접속 (혹은 UI 개발 모드로 Index.html 직접 오픈)

### 4.2 주요 디렉토리 트리
```text
C:\PJT_NEW\
 ├── app.py                     # Backend 서버 진입점 및 검색 로직 컨트롤러
 ├── search_ui.html             # 메인 베이스 HTML 마크업
 |
 └── frontend/                  # 정적 리소스 파일
      ├── html/apps/
      │    ├── fileSearch.html  # 파일 검색 기능 렌더링 HTML 조각
      │    └── link.html        # 즐겨찾기 링크 렌더링 HTML 조각
      ├── css/
      │    └── style.css        # 스타일시트 정의서 (다크테마, 그리드, 애니메이션)
      ├── js/
      │    └── main_js.js       # 비동기 통신, 라우팅, 데이터 바인딩 로직
      └── img/
           └── logo.png         # 애플리케이션 로고 이미지
```

## 5. 향후 확장성 고려사항 (Roadmap)
- **AI 시멘틱 분석 고도화:** 현재 패턴 매칭 및 OCR 수준으로 구현된 `content` 검색을 진정한 딥러닝 임베딩 방식(예: Gemini, OpenAI API) 또는 Vector DB 연동으로 고도화하여 맥락 기반 검색 지원 (예: "결산 내역" 검색 시 "재무제표" 파일 매칭)
- **검색 속도 최적화:** 실시간 폴더 순회(`os.walk`) 방식에서 백그라운드 스케줄러를 통한 Indexing Database 구축 (SQLite 등) 마련 필요
